Profile Page

Technical Description

The profile page is structured with semantic HTML elements such as headers, sections, and div containers to clearly organize content like user information, skills, goals, and contact forms. CSS Flexbox is used for responsive layouts and proper alignment, while external resources (Google Fonts and Font Awesome) enhance typography and icons.

Accessibility

Accessibility is ensured by using descriptive alt text for images, correctly associated form labels, and semantic HTML that aids screen reader navigation. Icons are paired with text and include appropriate aria attributes, ensuring that all users can understand the content.

SiteMap Page

Technical Description

The sitemap page uses semantic HTML to organize content around a dynamic SVG that visually maps the website’s structure. The SVG includes interactive elements—clickable rectangles and lines—that represent different pages and their relationships. CSS is applied to style these SVG elements and ensure a consistent, responsive design.

Accessibility

Accessibility is enhanced by adding descriptive text and using attributes like tabindex on interactive SVG elements, allowing keyboard navigation. Additionally, images include meaningful alt text to support screen readers.

Content Page

Technical Description

The content page is built with semantic HTML to structure informative sections on marine biodiversity, including introduction, coral reefs, whale sharks, mangrove ecosystems, and marine protected areas. CSS is used to style these sections, ensuring clear typography, responsive layouts, and engaging visual effects like a scroll progress bar and animated bubbles. Additionally, a floating section indicator helps users quickly navigate to different content parts, and a "Go to Top" button is provided for smoother navigation.

Accessibility

Accessibility is enhanced through the use of descriptive alt text on images and well-structured heading elements that support screen reader navigation. Interactive elements, such as the floating section links and the "Go to Top" button, are designed with keyboard accessibility in mind, ensuring that users with different abilities can easily access and navigate the content.

Challenges and Lessons Learned

During development, integrating multiple pages with distinct layouts (profile, sitemap, content) posed challenges, particularly with conflicting CSS rules and ensuring consistency in design. For example, resolving layout conflicts required careful renaming of classes and more specific selectors. We also encountered issues with accessibility, such as empty headings or inadequate alt text, which were resolved by adding fallback content and proper aria attributes. Overall, these challenges reinforced the importance of planning, code validation, and adopting best practices for responsive and accessible web design.

Compliance

All pages were built following Janet regulations governing web page publication. We ensured that our HTML is valid, our CSS is well-structured, and our interactive elements are accessible via keyboard navigation. The pages adhere to standards for semantic markup, proper alt text for images, and clear, logical content hierarchies. By meeting these requirements, we demonstrate a strong understanding and practical application of web compliance regulations.
